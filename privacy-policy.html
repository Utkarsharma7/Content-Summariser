Privacy Policy

NotebookLM PDF Summariser — Last updated: March 27, 2026

What This Extension Does

NotebookLM PDF Summariser is a Chrome extension that detects PDF files open in your browser and lets you generate summaries, presentations, podcasts, quizzes, or chat conversations from them using Google NotebookLM.

Data We Access

  • PDF content: When you choose to summarise a PDF, the extension reads the PDF from the active tab and sends it to a backend API server for processing.
  • Tab URLs: The extension reads the URL of your active tab to detect whether it contains a PDF.
  • Google cookies: The extension reads your Google session cookies (via chrome.cookies API for the .google.com domain) to authenticate with Google NotebookLM on your behalf. These cookies are sent to the backend API server with each request so it can make NotebookLM API calls using your Google account. Cookies are not stored on the server — they are only held in memory for the duration of your active session.
  • Extension settings: Your preferences (auto-detect toggle, backend API URL) are stored locally in chrome.storage.sync.

Where Your Data Goes

  • PDF files are sent to the backend API server whose URL is configured in the extension settings. By default this is a server you operate yourself.
  • The backend server forwards PDFs to Google NotebookLM for processing. Google's own privacy policy applies to data processed by NotebookLM.
  • Generated output files (reports, slides, audio, quizzes) are downloaded directly to your computer.

Data Storage & Retention

  • The extension does not store any PDF content, summaries, or personal data persistently.
  • PDF files and generated artifacts are held in temporary memory on the backend server only for the duration of processing, then deleted automatically.
  • No analytics, tracking, or telemetry data is collected by the extension.

Third-Party Services

  • Google NotebookLM: Used for AI-powered content generation. Subject to Google's Terms of Service and Privacy Policy.

Permissions Explained

  • tabs / activeTab: To detect PDF tabs and inject the chooser UI.
  • scripting: To display in-page toast notifications, the output chooser modal, and the chat widget.
  • downloads: To save generated files to your computer.
  • storage: To save your extension preferences.
  • cookies: To read Google session cookies for NotebookLM authentication.
  • Host permission (*.google.com): Required to read Google cookies for authentication.
  • Host permissions (optional): Requested when you enable auto-detect, so the extension can read PDFs from any website.

Your Choices

  • You can disable auto-detection at any time from the extension popup.
  • You can revoke optional host permissions from chrome://extensions.
  • You control which backend server your data is sent to.
